Phase 0: Requirements Specification

  • I need to use Javascript to build a webpage with the DOM instead of HTML.
  • This Javascript webpage will have a calculator that can create objects in the DOM.
  • The user interacting with the webpage will also be able to destroy these DOM objects.

Good Solution

  • It can take all inputs into the calculator, including negative inputs and dividing by zero.
  • All of the operations can be deleted in any order. It doesn't matter if it's at the top, bottom, or middle.
  • The user interacting with the webpage will be able to follow all of the prompts necessary.
  • It also won't crash when you pass in a string.

Challenges

  • Understanding Javascript will be a challenge for sure. I've never used Javascript before, so I will definitely need to do my research.
  • I'll need to learn the syntax for changing the div color dynamically at the users will.

Knowns

  • Creating the divs shouldn't be too bad. I've done that before on a lot of different assignments.
  • Evaluating expressions also should be pretty easy.

Phase 1: System Analysis

Data

  • There will be data created in the JavaScript. That will be the only place that the data is coming from to create the HTML.
  • There will be some data coming from the stylesheet to help out with some of the divs.

Output

  • It will be a calculator that is fully created by JavaScript.
  • This calculator can evaluate expressions that are put into it.
  • It will also be able create and remove divs of the expressions that were completed.

Algorithms

  • CreateDiv - We create the div that is going to be used to show the expression.
  • CreateErrorDiv - Similar to the CreateDiv, but it is going to be used to create the divs that come up when an error is found.
  • Evaluating Expression - It just takes the 2 input values and an operand and computes the end value.
